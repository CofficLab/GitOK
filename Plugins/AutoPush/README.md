# AutoPush 插件

自动推送当前分支到远程仓库的 GitOK 插件。

## 功能特性

- 🚀 **自动推送**：当切换到启用了自动推送的分支时，自动将本地提交推送到远程仓库
- 📋 **分支级配置**：可以为每个项目的每个分支单独配置是否启用自动推送
- 💾 **持久化存储**：配置会被保存，下次打开应用时自动恢复
- 🔔 **状态栏图标**：在状态栏显示图标，绿色表示已启用，灰色表示已禁用
- 📊 **配置管理界面**：点击状态栏图标可打开配置界面，管理所有已配置的项目分支

## 使用方法

### 启用自动推送

1. 打开 GitOK 应用
2. 确保状态栏显示了 AutoPush 图标（箭头向上圆圈图标）
3. 点击状态栏图标，打开配置界面
4. 在"当前项目"部分，找到你想要启用自动推送的分支
5. 打开"启用自动推送"开关

### 配置界面

配置界面分为两个部分：

#### 当前项目
显示当前选中项目的信息，包括：
- 项目名称和路径
- 当前分支名称
- 自动推送开关
- Git 仓库状态提示

#### 已配置的项目分支
显示所有已配置自动推送的项目分支列表，包括：
- 项目名称和分支名称
- 启用/禁用状态
- 最后推送时间
- 快捷操作（切换开关、删除配置）

### 工作流程

1. 当用户切换到某个分支时，插件会自动检查该分支是否启用了自动推送
2. 如果启用了自动推送，并且有未推送的提交，插件会自动执行 `git push`
3. 推送完成后，会更新最后推送时间
4. 如果推送失败，会显示错误提示

## 技术实现

### 文件结构

```
Plugins/AutoPush/
├── AutoPushPlugin.swift          # 插件主类，实现 SuperPlugin 协议
├── AutoPushService.swift         # 自动推送服务，监听分支变化并执行推送
├── AutoPushSettingsStore.swift   # 配置存储，管理项目分支的自动推送设置
├── AutoPushStatusIcon.swift      # 状态栏图标视图
└── AutoPushConfigView.swift      # 配置管理界面视图
```

### 核心组件

#### AutoPushPlugin
- 实现 `SuperPlugin` 协议
- 通过 `addStatusBarTrailingView()` 在状态栏添加图标
- 支持用户通过设置界面启用/禁用插件

#### AutoPushService
- 单例服务，监听 `projectDidChangeBranch` 通知
- 使用 Combine 监听 `DataProvider.$project` 变化
- 检查配置并执行自动推送逻辑

#### AutoPushSettingsStore
- 使用 UserDefaults 持久化存储配置
- 管理项目路径 + 分支名称的唯一配置
- 提供配置的增删改查方法

#### ProjectBranchAutoPushConfig
- 配置数据模型
- 包含项目路径、分支名称、启用状态、最后修改时间、最后推送时间等信息

## 注意事项

1. **仅在有未推送提交时执行**：如果本地和远程同步，不会执行推送操作
2. **需要远程仓库**：如果没有配置远程仓库（origin），自动推送不会执行
3. **错误处理**：推送失败时会显示错误提示，不会影响其他操作
4. **线程安全**：所有 Git 操作都在后台线程执行，UI 更新在主线程进行

## 开发调试

### 查看详细日志

在 `AutoPushService.swift` 中设置 `verbose = true` 可以查看详细日志输出：

```swift
private let verbose = true
```

### 禁用自动推送服务

在开发过程中，如果需要临时禁用自动推送服务，可以在 `RootView.swift` 中注释掉注册代码：

```swift
// AutoPushService.shared.register(dataProvider: self.git)
```
