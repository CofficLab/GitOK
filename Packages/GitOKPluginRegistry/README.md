# GitOKPluginRegistry

> Auto-generated plugin registry for GitOK.

## Overview
This package aggregates all enabled GitOK plugins into a single registry. It is automatically generated by the build-phase script (`Scripts/generate_plugin_registry.sh`) and should not be edited by hand.

## How it works
1. During the build, `generate_plugin_registry.sh` scans `Plugins/` for packages with active policies (`.optIn`, `.optOut`, `.alwaysOn`).
2. It generates `GeneratedPluginRegistry.swift`, which imports all active plugins and registers their adapters.
3. The main app calls `GeneratedPluginRegistry.registerDefaultAdapters` to load them at startup.

## Structure
- `Package.swift`: Package manifest.
- `Sources/GitOKPluginRegistry/GeneratedPluginRegistry.swift`: Auto-generated registry code.
